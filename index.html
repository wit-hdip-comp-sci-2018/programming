This section details the assignments for module

1a: Introduction to Processing

Through Processing (with Java), you will learn how to draw and format basic shapes, use colour (Grayscale and RGB), solve basic syntax errors and comment your code.

1b: Animating your Drawings

Start animating simple drawings, whilst also learning about Processing's system variables, Java's primitive data types and performing calculations on those data types.

2a: Selection and Events

Use selection statements (if), relational operators, logical operators, variables, mouse events and key events to further animate your drawings.

2b: Iteration (Loops)

Write progams that engage in repetitive behaviour (using for and while loops). We will also look at variable scope and compound assignment statements.

3a: Methods (Part 1)

Learn how to use mouse event methods and also learn how to write your own, bespoke methods.

3b: Methods (Part 2)

Evolve a deeper understanding of methods. Learn more about Strings and the methods that you can use on them.

4a: Classes (Part 1)

Learn more String methods and how the characters are indexed within the Strings. Start writing your own, bespoke classes.

4b: Classes (Part 2)

Continuing with classes, focus on Encapsulation techniques whilst adding methods and field validations.

5a: Arrays

We will introduce the concept and syntax of primitive arrays. Then we will use JOptionPane to read input into these arrays.

5b: Arrays and Classes

We will continue building more classes and using Arrays.

6: Pong

Start building the Game of Pong and evolve it to include Player details, tournaments, average scores, collision detection, etc.

7: IntelliJ and Basic I/O

Start using IntelliJ instead of Processing. As part of this transition, we will introduce you to the Java Virtual Machine (JVM) and I/O using Scanner.

7b: Grouping Objects

We will do a recap of Primitive Arrays and move onto the Collections Framework, starting with ArrayLists.

8a: ArrayList and Menu Driven Apps

We will continue with ArrayList Collections whilst focusing on developing Menu Driven applications.

8b: Persistence (CRUD)

We will start looking at persisting data beyond the lifetime of the application. Covering the CRUD (Create, Read, Update, Delete) process is the first step in persistence.

9a: Persistence (XML & Exceptions)

Having covered the CRUD process, we will now persist our application data to an XML file. As part of this process, we will cover handling of potential errors (Exceptions).

9b: Collections (Map and Set)

We have studied ArrayLists. We will now look at two other Collections, Map and Set. We will look at them in the context of the Tech Support System App.

10a: Inheritance and Polymorphism

We will investigate two Object-oriented concepts: Inheritance (the is-a relationship) and Polymorphism (many shapes!).

10b: Abstraction and Debugging

We will now look at a third Object-oriented principle, Abstraction. We will also cover Debugging in the IntelliJ IDE.

11: TDD and JUnit

We will use Test Driven Development (TDD) approaches in our app development. In particular, we will use the JUnit Framework to automate our testing.

12: Interfaces

In this final section, we will do a recap of abstraction in Java prior to covering Interfaces, a way of realising abstraction.
